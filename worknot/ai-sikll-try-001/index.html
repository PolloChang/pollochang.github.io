2. 關係詳細說明
#

這 7 張表可以被分為三大類：

A. 動化主檔表 (Dimension Tables)
#

  1. ANIME_DATASET_2023 與 ANIME_FILTERED：
    • 關聯欄位：ANIME_ID
    • 關係類型：一對零或一 (1-to-0/1)。
    • 說明：ANIME_DATASET_2023 為 2023 年完整的動畫清單（24,905 筆），而 ANIME_FILTERED 為經過篩選過後的動畫子集（14,952 筆）。兩者皆以 ANIME_ID 為主鍵（Primary Key）。

B. 使用者主檔表 (Dimension Tables)
#

  1. USERS_DETAILS_2023：
    • 關鍵欄位：MAL_ID (主鍵) 與 USERNAME (邏輯唯一鍵)。
    • 說明：存放使用者的基本輪廓資料（731,290 筆）。其主鍵 MAL_ID 對應至其他互動資料表中的 USER_ID，而 USERNAME 亦與其他表中的使用者帳號名稱對應。

C. 使用者與動畫互動資料表 (Fact Tables / Many-to-Many)
#

這三張表記錄了使用者對動畫的評分，皆為多對多關係：

  1. USERS_SCORE_2023 (2,432 萬筆)：
    • 關聯鍵：
      • USER_ID 關聯 USERS_DETAILS_2023.MAL_ID
      • USERNAME 關聯 USERS_DETAILS_2023.USERNAME
      • ANIME_ID 關聯 ANIME_DATASET_2023.ANIME_ID
  2. USER_FILTERED (1.09 億筆)：
    • 關聯鍵：
      • USER_ID 關聯 USERS_DETAILS_2023.MAL_ID
      • ANIME_ID 關聯 ANIME_DATASET_2023.ANIME_ID
    • 說明：這張是體積極小但筆數極多（1.09 億筆）的評分純對應表，僅包含 USER_ID, ANIME_ID, RATING 三個欄位。
  3. FINAL_ANIMEDATASET (3,263 萬筆)：
    • 關聯鍵：
      • USER_ID / USERNAME 關聯 USERS_DETAILS_2023
      • ANIME_ID 關聯 ANIME_DATASET_2023.ANIME_ID
    • 說明：一個已經過反正規化（Denormalized）的大型評分資料集，裡面同時冗餘了動畫的部分資訊（如 TITLE, TYPE, SOURCE, SCORE, GENRE 等）以利快速查詢。

D. 孤立資料表 (Isolated Table)
#

  1. GOV_DATASET_REGISTRY (3.8 萬筆)：
    • 主鍵：OBJID
    • 說明：這張表主要記錄政府開放資料集的 metadata（註冊資訊、聯絡人、下載連結等），與上述的動畫或使用者資料完全無關。

3. SQL 關聯範例與效能建議
#

由於三個巨型資料表（USER_FILTERED, FINAL_ANIMEDATASET, USERS_SCORE_2023）沒有索引，在進行關聯時務必特別小心效能。

範例一：查詢特定使用者 (ID=1) 評分過的動畫名稱
#

使用 USER_FILTERED 與 ANIME_DATASET_2023 關聯：

SELECT 
    f.USER_ID,
    f.RATING AS USER_RATING,
    a.NAME AS ANIME_NAME,
    a.SCORE AS GLOBAL_SCORE
FROM USER_FILTERED f
JOIN ANIME_DATASET_2023 a ON f.ANIME_ID = a.ANIME_ID
WHERE f.USER_ID = 1; 
-- ⚠️ 注意：f.USER_ID 目前無索引，此查詢將會進行 1.09 億筆資料的全表掃描 (Full Table Scan)，極為緩慢。
